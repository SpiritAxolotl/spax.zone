How did we get here?

Hi! I'm Spax, the creator of The Horse Button™. You might be asking yourself: what the hell did you make and why? And to that, I can at least try to explain everything. (If you want the tl;dr, click here to jump to the bottom)

The beginning

So, in about 2023, I discovered a site called every.horse, a periodically-updating list cataloging every registered .horse domain. The creator of this website has his own write-up on why his site exists too. I clicked on a few links on the site and I vividly remember stumbling across this primitive pixel art RPG that ran in the browser. I thought it was neat and then left the site and resumed whatever it was I was doing.

About a year later, in February 2024, I wanted to go back to this site because someone's Discord avatar reminded me of one of the character portraits in the game, but I could not find anything in my browsing history, and have been unable to find this silly game ever since. I then felt determined to find this game. I had forgotten the domain for every.horse, so I myself requested the zone file for the .horse domain, and pretty soon after, I had a list of roughly 3000 .horse domains in a text file. I sent it to one of the Discord servers I was in, and I soon decided to make a horse button, where upon pressing it, you'd be sent to a random registered .horse domain, thinking the journey to find this game would be more fun as a random button.

Nerd emoji

With the idea devised, I—in the middle of the night, having to wake up at 7am for school the next day—cobbled together a prototype that had a button with the intended functionality. The page was completely devoid of css, and I had converted the text file of .horse domains into a json file stored on my repository. The same day (at night though), I polished the site a bit more with some css, and added a bit of noscript code the next day.

Reception

With the site still in an early prototype, I sent it to some of my friends and a couple of servers I was in, with the disclaimer of "please click at your own risk Lol". People seemed to really like it! For a good few hours, I and others had spent some time clicking the button and sending the highlights to each other. Even after the first week, there had just been times in the rest of 2024 where I'd bring up the horse button in a voice chat or group chat, followed with "what is that??", and having another session of sending highlights. Many great memories have been made.

Nothing can be perfect

Somehow, I keep going back to this site. The original goal was to find one particular game in a sea of domains, but the project had become something more than that. I wanted to make the site better than it already was. Why? I've learned about a lot of workflows over the years, from game developer talks, friends, etc. What I have learned about myself, is that my ideal workflow is: getting something out, and polishing it over time as my skills develop. And this project seems to be no exception. I've had plenty of ideas to improve the site, Useful settings, silly (but subtle) references, better UI/UX practices, etc.

Nerd emoji 2

In October 2024, I decided to work on the website yet again. I had just moved my site's content from Github Pages to Cloudflare Pages, which gave me a lot more flexibility upfront (most notably build instructions). I split the horse.html file into typical index.html/styles.css/script.js files. The json file with all the horse domains I had requested myself was already out of date, so I made an extra get-horselist.js file that would be run every time I pushed to my repository too, since I was learning how to use node.js at the time. It scraped the every.horse site (that I had finally remembered), which a part of me felt bad about, but I decided to not do anything about because I didn't know what else to do. I had also added a hostname preview when the button was hovered over, so that part of the fun was seeing what site you would go to before being greeted with its page contents.

The nagging thought that more could be done

All-in-all, this was a pretty solid state of the website, but there was something nagging at me, telling me that there was a better way to approach the website; that scraping a website every time a push was made was inefficient and clumsy; that it's not fun to manually sift through a bunch of blank and ip-less sites to find something that actually has an interesting webpage. I had been a high school graduate for almost a year, and had taken up some jobs in my gap year before going off to college, and work had drained my energy most days. Once my life had become a little less busy, I started to tinker with the horse button and made large changes to the get-horselist.js (now-renamed horse-rotate.js), before my life became Very busy again and couldn't actually work on it anymore. But then a month later, I was finally able to put aside time and make the changes I wanted to.

You guessed it! Nerd emoji 3

This section of the story is still in progress, as I am currently working on the horse button's entire refactor. A few things to note: horse-rotate.js now scrapes every.horse every 24 hours, updating this dynamic list, and pings one of the .horse domains before doing the next one in the list every 5 minutes. As for the scraping, I've let the owner of the site, Ryan Bateman, know that I've been doing this, and requested a sister json file that others (me) could fetch from without it necessarily being scraping at that point. He responded that he was currently on a trip, but he'd be happy to just share the python script he wrote to request the zone file, once he's back. This means that hopefully I'll just be able to have a fresh source for the every-horse list! I'm also planning on adding some quality of life settings, and if I'm able to figure out how to do it, possibly have a rating system for all horse domains? Might be fun, but we'll see how difficult that is to implement. The big feature that I would like to have is filtering for the different sorts of websites (ip-less, parked, redirects, blank, and all that's left). So far, just the ip-less sites have a filtered list, but I'll add the rest soon.
